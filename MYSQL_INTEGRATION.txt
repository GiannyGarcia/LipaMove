LipaMove — MySQL Workbench + Node API
=====================================

Why a server?
-------------
Browsers cannot connect to MySQL directly. The Node app (server/index.js) uses the mysql2 driver to talk to MySQL and exposes JSON APIs under /api/*. The same server also serves index.xhtml, data.xml, and assets so fetch("data.xml") and fetch("/api/...") share one origin (no CORS issues).


MySQL Workbench — create the database (recommended)
-----------------------------------------------------

Prerequisites: MySQL Server 8.0+ installed and running (Windows: Services → MySQL, or MySQL Installer).

1) Open MySQL Workbench. Under MySQL Connections, open your local instance (e.g. "Local instance MySQL80") and enter your root password when prompted.

2) One-shot setup (easiest):
   - Menu: File → Open SQL Script…
   - Select: database/lipamove_full_setup.sql
   - Click the lightning bolt icon (Execute) or Ctrl+Shift+Enter.
   - In the Output panel, confirm there are no errors.

3) Confirm:
   - In the left Schemas panel, click Refresh. You should see database `lipamove`.
   - Optional: run in a new query tab:
     USE lipamove;
     SHOW TABLES;
   You should see: operators, otp_codes, routes, route_waypoints, sessions, users, vehicle_positions, vehicles.

Alternative: run scripts separately
-----------------------------------
- server/schema_auth.sql — users, otp_codes, sessions (create DB first; see comments at top of file).
- lipamove_schema.sql — fleet tables + sample seed data.
The combined file database/lipamove_full_setup.sql replaces running both in order.


Configure the Node server
-------------------------

1) Copy server/.env.example to server/.env

2) Edit server/.env:
   MYSQL_HOST=127.0.0.1
   MYSQL_PORT=3306
   MYSQL_USER=root
   MYSQL_PASSWORD=<your MySQL root password>
   MYSQL_DATABASE=lipamove
   DEV_RETURN_OTP=true

   DEV_RETURN_OTP=true includes the OTP in API JSON for local demos only. Turn off in production.


Start the API (Windows)
-----------------------

   cd server
   npm install
   npm start

Open the app in the browser (same host as the API — not file://):

   http://127.0.0.1:3000/index.xhtml

Test database connectivity:

   http://127.0.0.1:3000/api/health

Expect JSON like: {"ok":true,"db":"connected"}


Auth flow (prototype)
-----------------------
- Sign up: POST /api/auth/register → row in otp_codes; dev build may return devOtp in JSON.
- Verify signup: POST /api/auth/verify-signup with phone + 6-digit code → session token.
- Log in: POST /api/auth/login → OTP in otp_codes; verify with POST /api/auth/verify-login-otp.
- Profile: GET /api/me with header Authorization: Bearer <token>.


Production notes
----------------
- Replace DEV OTP display with SMS (Twilio, etc.) and remove devOtp from responses.
- Use HTTPS, strong secrets, rate-limit /api/auth/*, and hash or encrypt OTP storage as required by your policy.
- Create a dedicated MySQL user with privileges only on database `lipamove` instead of root.
